## Recommended actions

Prioritised:

1. **No urgent action, and nothing production-facing.** All 10 HIGHs and the new LOW are in
   the dev-only Appium e2e tree. The shipped wallet is unaffected.
2. **Track the Appium major** — `appium-uiautomator2-driver ^8.1.0 → 4.2.9` (the package
   renumbered; npm reports it as semver-major). This clears all 10 HIGHs plus the
   `body-parser` LOW in one move. Schedule as a deliberate devDependency upgrade with an
   Android e2e smoke run; do **not** apply via `npm audit fix --force`.
3. **Apply trivial patch bumps** in a routine maintenance PR (not this report-only run):
   `vite 8.1.4 → 8.1.5`, `@capacitor/* 8.4.1 → 8.4.2` (+ `app` 8.1.1, `local-notifications`
   8.2.1). Low risk, keeps the native bridge current.
4. **Track the Ledger major** (`@ledgerhq/hw-app-eth` → 6.40.3) as a tested upgrade — it
   clears the entire `@ethersproject/*` v5 sub-tree. Gate behind a hardware-signer regression
   pass (semver-major).
5. **Monitor the Trezor chain** — no upstream fix for `tiny-secp256k1`/`elliptic` under
   `@trezor/utxo-lib` today; re-check next weekly run.
6. **Do NOT** bulk-bump `@noble/*` / `@scure/*` to v2 as a currency chore. They are not
   vulnerable, and v2 is a breaking major touching the seed/derivation/signing core — migrate
   only with a dedicated branch, strict TDD, and full BIP-32/39/SLIP-0010 spec-vector
   re-verification, per the project's no-blind-security-changes rule.
