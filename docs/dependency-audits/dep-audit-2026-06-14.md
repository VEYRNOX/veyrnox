# Dependency audit — 2026-06-14 (raw snapshot)

Companion note for `dep-audit-2026-06-14-raw.json`, which is unprocessed
`npm audit --json` output rather than a written report like the other files here.

The JSON was found untracked in the repo root on 2026-07-19 (matched by the
`audit.json` entry in `.gitignore`, so it had never been committed) and was moved
here rather than deleted — it is a non-regenerable record of the dependency
posture a month earlier.

## What it captured

`0 critical / 7 high / 11 moderate / 0 low` across 18 vulnerable packages.

**High (7):** `esbuild`, `flatted`, `lodash`, `minimatch`, `picomatch`, `rollup`,
`vite`*

**Moderate (11):** `@solana/web3.js`*, `ajv`, `brace-expansion`, `dompurify`,
`ethers`*, `jayson`, `postcss`*, `react-router`, `react-router-dom`*, `uuid`, `ws`

`*` = direct dependency.

Substantively these were build-toolchain advisories (`esbuild`, `rollup`, `vite`,
`picomatch`, `minimatch`, `flatted`, `postcss` — ReDoS, path traversal, arbitrary
file write) plus application-layer ones, of which `dompurify` was the heaviest at
eight distinct XSS / prototype-pollution advisories.

## Delta to 2026-07-19

| | 2026-06-14 | 2026-07-19 |
|---|---|---|
| critical | 0 | 0 |
| high | **7** | **0** |
| moderate | **11** | **0** |
| low | 0 | 18 |
| total | 18 | 18 |

**The matching totals are coincidence.** The two sets share no packages. Every
2026-06-14 finding is resolved; all 18 findings on 2026-07-19 root in a single
advisory — `elliptic`, "Uses a Cryptographic Primitive with a Risky
Implementation" — which is one root plus 17 transitive dependents, and did not
appear in the 2026-06-14 set at all.

Read as a trend: 7 high and 11 moderate cleared, replaced by one low-severity
advisory with no upstream fix at any version. `elliptic` is pinned to its latest
release (`^6.6.1`) via `package.json` `overrides` and is not on the wallet's
signing path — `src/wallet-core/` uses `@noble`/`@scure` and ethers v6; it reaches
the tree only through hardware-wallet transport code (`@trezor/connect-web` and
`@ledgerhq/hw-app-eth`). It is recorded as an accepted residual in the daily audit
task, scoped by package name, and is still counted and disclosed on every run.

## Honest scope

`npm audit` reports advisories against the declared dependency tree. It is not a
statement about exploitability in this application, and neither snapshot was
manually triaged per-finding at the time it was taken. The 2026-06-14 figures are
reproduced from the JSON as-is.
