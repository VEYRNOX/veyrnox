`@ledgerhq/hw-app-eth` — they are **not** the wallet's signing path, which uses ethers v6
+ `@noble`/`@scure`.

## Moderate / Low (summary)

18 **low** advisories, all one root cause (`elliptic` GHSA-848j-6mx2-7j84):

- **Ledger chain** (`@ledgerhq/hw-app-eth`, direct dep): `@ethersproject/{abi,
  abstract-provider, abstract-signer, hash, signing-key, transactions}`,
  `@ledgerhq/evm-tools`. These are ethers-**v5** internals bundled by the Ledger SDK, not
  the app's signing code. `fixAvailable` = `@ledgerhq/hw-app-eth@6.40.3` (**semver-major**).
- **Trezor chain** (`@trezor/connect-web`, direct dep): `@trezor/{connect, blockchain-link,
  blockchain-link-types, utxo-lib}` → `tiny-secp256k1` → `elliptic`; plus
  `crypto-browserify` → `browserify-sign` / `create-ecdh` → `elliptic`. `fixAvailable` =
  **false** (no non-breaking upstream release).

Exposure is limited: `elliptic` is exercised only inside vendor hardware-wallet SDK code
paths, and the advisory is a low-severity risky-primitive class issue, not a remote-exploit
or key-leak advisory.

   vulnerable, and v2 is a breaking major touching the seed/derivation/signing core —
   migrate only with a dedicated branch, strict TDD, and full BIP-32/39/SLIP-0010 spec-vector
   re-verification, per the project's no-blind-security-changes rule.
