## E2E Web Flows

After the F-001 fix: **57 passed / 1 failed / 3 skipped** (61 tests). 4 new demo-isolation specs all pass. Throwaway seed moved to git-ignored `.env.test`, loaded via dotenv in `playwright.config.ts`.

- **F-001 (CRITICAL, fixed)** — `@revenuecat/purchases-capacitor` unresolved by Vite dev server, blanking the app and breaking 18 of 19 E2E failures. Fixed with a no-op web stub alias (all callers already guard on `isNative()`).
- **F-002 (LOW, fixed test-only — was HIGH)** — `duress-decoy-routing.spec.js` intermittently couldn't find `HIDDEN WALLET` after Emergency-PIN unlock. **Diagnosed via live trace 2026-07-11: root cause is a test-timeout flake, not a regression.** The Emergency-PIN unlock measures **8.7–10.0 s** (run 1 = 9996 ms pass, run 2 = >10000 ms fail, run 3 = 8712 ms pass) — it takes the failure→deniability path (4–5 Argon2id KDFs at 192 MiB + the 2000 ms equalizer), landing right on the test's 10 s ceiling and flaking at the margin. Decoy routing is **correct** every run (`isDecoy` flips, `HIDDEN WALLET` renders, decoy address differs); swallowed-error and real-regression both ruled out. **Fix:** assertion timeout bumped 10 s → 30 s, 3/3 stable. Landed on `main` via PR #822 (squash-merged as `eb41102a`, bundled with the other QA fixes); the standalone PR #823 carrying the identical change was closed as superseded by #822. Selector and wallet-core untouched. → downgraded to P2.
- **F-003 (MEDIUM, open)** — `frame-ancestors` CSP in a `<meta>` element is silently ignored by browsers; no clickjacking protection. Must move to an HTTP response header. → P2.
- **F-004 (LOW, open)** — Send form unreachable at `/send?demo=1` without vault state; E2E marked `test.skip`. Needs a pre-seed fixture. → P2.
