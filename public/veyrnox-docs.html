Product Vision

A self-custody wallet that protects your keys — even under duress

Veyrnox is a multi-chain cryptocurrency wallet for people who take self-custody seriously. One HD seed derives all accounts. Keys never leave the device. No cloud, no custodian, no server that can be subpoenaed. The app adds coercion resistance — duress PINs, hidden wallets, panic wipe — for users whose threat model extends beyond phishing.

Web + iOS + Android via Capacitor 8. Free tier covers core wallet operations. Safety Plus ($5.99/mo or $49.99/yr via RevenueCat IAP) unlocks advanced security, analytics, and coercion-resistance features.

Security (Mixed Tiers)

Hardware KEK BUILT PLUS

iOS: Secure Enclave ECIES P-256 (non-extractable). Android: StrongBox HMAC-SHA256. KEK = HKDF-SHA256(H || C). Auto-enroll on all wallet entry paths. v1/v2/v3 protocol upgrade path.

Duress PIN BUILT PLUS

Separate AES-GCM encrypted decoy vault. Constant-time unlock resolution. Decoy balance funded on-chain. Face ID binding. Zero backend calls (I3). Visually identical to real session.

Stealth Wallets BUILT PLUS

Hidden wallets in fixed-size chaff slot pool. Real and chaff slots are indistinguishable AES-GCM ciphertext. HKDF-SHA256 secret-derived placement. Opt-in balance checks with privacy warning.

Panic Wipe BUILT PLUS

Irreversible destruction of ALL local key material. Two triggers: panic PIN at unlock (no confirmation), in-app guarded wipe (type "WIPE"). Clears IndexedDB + localStorage.

RASP BUILT

Runtime tamper detection. Browser probes + native probes + Play Integrity + App Attest. Conditions → tiers (ALLOW/WARN/BLOCK). Deniability-safe (identical response for real/decoy). Pre-sign gate integration. Fail-closed.

Risk Scoring LIVE

8 signal detectors: fresh recipient, unlimited approval, fresh spender, address poisoning, ENS mismatch, dust input, calldata mismatch, value anomaly. Composite score applied to send + WalletConnect.

Transaction Simulation LIVE

Local-first eth_call dry-run. No third-party scoring. Detects reverts, unlimited approvals, known-bad recipients, unverified contracts, address poisoning, large outflow, anomaly heuristics.

Biometric Unlock BUILT

Face ID / Touch ID via @aparajita/capacitor-biometric-auth on native. WebAuthn on web. Stale biometric cache clearance. Deniability-safe suppression in decoy sessions.

Passkey Unlock BUILT

FIDO2/WebAuthn passkey gate. Registration + assertion flow. Sign-count tracking. Parallel to biometric gate, not key custody.

2FA Gate BUILT

Three modes: password (PIN + action password, dual Argon2id), passkey (PIN + WebAuthn assertion), biometric (biometric-only step-up). 5-attempt cap with wallet lock. Method priority: biometric > passkey > password.

Anomaly Detection BUILT PLUS

Three local heuristics: unusual amount vs median history (10x), large to first-time recipient (50% of balance), approval-then-transfer pattern. Sigma-threshold math, no phone-home.

Token Approvals BUILT PLUS

View allowances with unlimited flag. Revoke-only (no approve UI — unlimited approvals are the #1 drain vector). Cross-checks on-chain decimals. Calldata self-verification.

Spam Token Filter BUILT

Pure deterministic classifier: URL in name/symbol, lure keywords, non-ASCII homoglyphs, unsolicited airdrop, worthless balance. Verified tokens bypass. User override wins.

Address Poisoning BUILT PLUS

Look-alike detection: same first/last 4 hex nibbles, different middle. Screens against user's history/address book. NULL and DEAD addresses flagged.

Audit Log BUILT PLUS

Opt-in encrypted vault blob. Type + timestamp only (max 100 entries). Primary-session only; no-op in decoy/hidden. Toggle + viewer + clear in Settings.

Session Manager BUILT

UserSession list/revoke. Auto-lock timeout (configurable). Session revocation guard in Layout locks wallet on revoke.

Security Dashboard LIVE

Aggregated posture view. Stat cards: unlimited approvals, flagged spam, flagged addresses. Feature rows: biometric/passkey/auto-lock/duress/stealth/panic/RASP status.

Design Principles

Deniability by default

No wallet count, wallet list, or wallet selector anywhere. Dashboard shows one wallet's balance. Stealth pool: never reveals configured state.

Plain-language risk

"This address was seen in a phishing attack" not "ADDRESS_POISONING_DETECTED." Every warning is actionable by a non-technical user.

Earned status

BUILT = code-complete. VERIFIED = on-chain txid confirmed on block explorer. Never aspirational.

Fail visible

"Hardware protection unavailable on web" not silently downgrading. Indeterminate balance shows "incomplete" marker, never silent $0.

Monospace = verifiable

IBM Plex Mono for addresses, amounts, txids. Labels never in mono. The type distinction itself is information.

No approve() UI

Unlimited token approvals are the #1 drain vector. Only revoke is exposed. The deliberate absence is a design decision, not a missing feature.

Data Architecture

On-device storage + minimal backend

Almost everything is on-device in IndexedDB and localStorage. The only server-side persistence is two Supabase PostgreSQL tables for referral tracking. No keys, balances, transaction data, or user identity ever reach the backend. External APIs are client-side only — no backend proxy.

Open Residuals
M-1
EVM key unzeroable (ethers v6) — JS immutable string holds the decoded seed in memory. ethers v6 returns the private key as a string that cannot be overwritten. Architectural limitation of the runtime; no fix available without replacing ethers.
M-6
iOS bridge H copy — Capacitor bridge serializes the hardware KEK factor H as a base64 NSString. NSString is immutable and cannot be zeroed in place. The decrypt path uses NSMutableData correctly; the bridge transit layer is the residual.
#1111
